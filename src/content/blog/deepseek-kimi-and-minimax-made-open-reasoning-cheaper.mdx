The 2025 open-weight story from China was not one model. It was a sequence of releases that made the cost curve feel unstable in a good way. DeepSeek-R1 set the tone in January. MiniMax-M1 followed in June with a large hybrid-attention reasoning model. Moonshot's Kimi K2 landed in July with a trillion-parameter mixture of experts model tuned hard for coding and agentic work.

DeepSeek-R1 mattered because it made reasoning feel less monopolized by closed labs. Even if a team did not ship it directly, it changed the reference point. Open reasoning models could be strong enough to force every paid model call to justify itself.

MiniMax-M1 was interesting for a different reason. The hybrid-attention pitch was about making long-context reasoning and test-time compute cheaper to run. The model was based on a large MoE line with tens of billions of active parameters, but the product point was efficiency under hard reasoning workloads. That is exactly where model cost hurts: long tasks, repeated attempts, and agents that need more than a one-shot answer.

Kimi K2 made the coding and agent angle louder. Moonshot described Kimi K2 as a 1T total parameter MoE model with 32B active parameters, trained with the Muon optimizer and optimized for agentic capabilities. The part I cared about was not the giant total number. It was whether a model like that could make high-quality coding and tool use cheaper enough to change routing.

Compared with the 2024 open-weight wave, these releases felt less like "local alternatives" and more like serious pressure on frontier pricing. Llama, Qwen, Gemma, and Mistral had already made open weights normal. DeepSeek, MiniMax, and Kimi made the reasoning and coding tier feel contestable.

I would still keep a skeptical eval harness. Cheap reasoning is only cheap if it does not create hidden review cost. A model that writes plausible code but misses repo conventions can be expensive in the hands of the reviewer.

The useful move is to route carefully: open reasoning for drafts, search, coding attempts, synthetic data, and batch work; closed frontier models where reliability, tool use, or policy still wins. The best stack is probably mixed.
