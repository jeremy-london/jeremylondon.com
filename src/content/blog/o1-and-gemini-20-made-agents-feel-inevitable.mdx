By the end of 2024, the model race had moved from "which chat answer is better" to "which model can work through a task." OpenAI's o1 launch in September and Google's Gemini 2.0 launch in December made that shift hard to ignore.

o1 was the cleaner conceptual break. OpenAI described a model family designed to spend more time thinking before responding, with stronger results on math, coding, and science tasks. Compared with GPT-4o, o1 was not trying to be the fastest multimodal assistant. It was trying to be the model you call when the answer needs deliberate reasoning. o1-mini made the tradeoff more interesting by offering a cheaper STEM-oriented version that could beat bigger general models on some reasoning tasks while losing on broad knowledge.

That mattered because it introduced a new product knob: thinking time. Previously, latency was mostly a cost to reduce. With reasoning models, latency could become part of the value. Some tasks deserve a slower answer if the model actually uses the time to check itself.

Gemini 2.0 made the agent story broader. Google emphasized native image and audio output, tool use, and agentic experiences like Project Astra, Project Mariner, and Jules. Compared with Gemini 1.5, the story moved from "the model can take a lot in" to "the model can act across tools and modalities." That is a different product problem.

The OpenAI and Google launches were not saying the same thing. OpenAI's o1 asked developers to separate fast assistant paths from hard reasoning paths. Google's Gemini 2.0 asked developers to imagine more tool-using, multimodal agents. Together they made a point that still feels current: chat is only one surface. The model is becoming a planner, a tool caller, a reviewer, and sometimes a bad operator that needs a leash.

My note at the time would have been: do not ship agents until you can inspect them. Reasoning time and tool use both create hidden state. The user sees an answer or an action, but the important question is what happened in between. What did the model consider? Which tools did it call? Which output was rejected? Which permission boundary stopped it?

Compared with the previous generation, these releases were good because they made harder tasks plausible. They were risky for the same reason. The more a model can do, the more the product has to explain.
