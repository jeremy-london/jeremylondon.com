Liquid AI's first LFM release in September 2024 was one of the small-model launches I wanted to pay attention to precisely because it was not just another miniature transformer story. The first lineup included a dense 1.3B model, a dense 3.1B model, and a 40.3B mixture of experts model.

The interesting part was the claim behind the family: build foundation models from a different architecture and optimize for deployment constraints instead of treating edge use as an afterthought. I am always cautious when a new architecture is pitched too cleanly, but I like the pressure it puts on the conversation. Small models do not have to be only weaker versions of large models.

Compared with Gemma and Phi, LFM felt less like a "small but strong" release and more like a "what should the model be shaped like if it is going to live near the device?" release. That is a different product question. On-device and low-latency work cares about memory, throughput, battery, privacy, and predictable behavior as much as a benchmark score.

The small model story got more interesting after that. Liquid kept pushing device-aware and vision-language variants, while the rest of the field also moved toward smaller specialists. That is the path I care about: not one local assistant that tries to be everything, but a set of local or cheap models that handle narrow work well.

The practical test is boring. Can it classify the thing quickly? Can it extract the field without sending private text away? Can it run in the product path without turning the UI into a spinner? Can it fail in a way the user understands?

LFM was worth writing about because it made model architecture feel connected to product placement. The model's shape and the user's device started to belong in the same conversation.
