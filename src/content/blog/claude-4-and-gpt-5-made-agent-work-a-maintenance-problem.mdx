Claude 4 in May and GPT-5 in August made agent work feel less like a demo category and more like maintenance work. That may sound less exciting than the launches, but it is where the real change landed for me.

Anthropic's Claude 4 launch put Opus 4 and Sonnet 4 directly into coding and long-running agent workflows. Opus 4 was pitched as a top coding model with sustained performance across complex tasks. Sonnet 4 was framed as a serious upgrade to Claude 3.7 Sonnet. Compared with Claude 3.7, the point was not only "more reasoning." It was longer focus, better coding, and a stronger case for handing the model a multi-step engineering task.

That changes supervision. A model that answers one hard question can be checked like an answer. A model that works through a repo for an hour needs a different review surface. You need artifacts, diffs, logs, checkpoints, and a way to stop the work when it starts widening beyond the goal.

GPT-5 was a different kind of consolidation. OpenAI made it the default in ChatGPT, replacing GPT-4o, o3, o4-mini, and GPT-4.1 for signed-in users, with routing that could apply deeper reasoning when needed. Compared with the April 2025 model menu, GPT-5 reduced some user-facing choice. The product said: stop making ordinary users choose between fast chat, coding, and reasoning every time.

That was unique. OpenAI was turning model routing into product behavior. The user could still ask for more thinking, but the default path became more automatic. The risk is that automatic routing can hide important tradeoffs. If a request gets a deeper reasoning path, does the user know? If it costs more or takes longer, does the product make that visible? If it chooses the wrong path, can the user recover?

My note in August would have been that agents need maintenance budgets. They need the same boring care as any other production system: regression tests, permission reviews, incident notes, rollback paths, and ownership.

Compared with the previous generation, Claude 4 and GPT-5 were good because they made longer work feel plausible. The unique risk was the same on both sides: the model could now produce enough activity that the human might confuse motion with progress.

The launch reaction I would trust is not "agents are here." It is "agents are now big enough to need ops."
