Spring 2025 was messy in the way useful periods usually are. Anthropic launched Claude 3.7 Sonnet in February as a hybrid reasoning model. Google launched Gemini 2.5 Pro Experimental in March with thinking on by default. OpenAI followed in April with GPT-4.1 for coding, instruction following, and long context, plus o3 and o4-mini for tool-heavy reasoning.

That is a lot of model news in a short window. The mistake would be treating it as one generic "models got smarter" moment. The providers were making different bets.

Claude 3.7 Sonnet's unique idea was hybrid behavior. It could answer quickly or use extended thinking, and API users could control the thinking budget. Compared with Claude 3.5 Sonnet, the jump was not only better reasoning. It was the productization of when to think longer. That made Claude feel more like a workflow model than a chat model.

Gemini 2.5 Pro pushed a similar theme from Google's side, but with long context and multimodality still central to the pitch. Compared with Gemini 2.0, Gemini 2.5 felt more serious for coding, math, science, and hard planning. The useful thing was not that it topped a leaderboard for a week. The useful thing was that Google had a credible answer for reasoning quality without abandoning the context and multimodal lane it had been building.

OpenAI's April releases split the work even more. GPT-4.1 was the practical coding and instruction-following upgrade, with a long-context story and smaller mini and nano variants. o3 and o4-mini were the reasoning models with tools: web, Python, image analysis, files, and more. Compared with o1, o3 looked like the stronger hard-problem model, while o4-mini made the speed and cost side more attractive.

The competitive takeaway was that "reasoning model" had stopped meaning one thing. Sometimes it meant controllable thinking. Sometimes it meant long-context planning. Sometimes it meant tool use. Sometimes it meant a cheaper model that was good enough to run many times.

My launch-week practice would be to split evals by decision type. Do not ask "which model is best?" Ask which model drafts correctly, which model debugs, which model refuses bad tool calls, which model notices missing evidence, and which model is cheap enough to use in a loop.

Spring 2025 made routing feel unavoidable. A single default model was becoming a lazy architecture decision.
